# Integration Challenge Resolutions

### Challenge A: ProtectSystem=strict and Configuration Files
**Conflict:** Setting `ProtectSystem=strict` mounts the root filesystem as read-only for service processes. Because environment files are placed in the `/opt/kijanikiosk/config` path, service units were immediately blocked from reading configuration keys.
**Resolution:** I incorporated the `ReadOnlyPaths` systemd directive mapped to `/opt/kijanikiosk/config`. This surgically punches a hole through the read-only sandbox, granting exact read requirements while maintaining strict system write boundaries.

### Challenge B: Health Check Ownership and ACLs
**Conflict:** The health check file is generated by provisioning logic running as root, causing the JSON artifact to inherit root-only privileges, while the monitoring subnet relies on reading it without elevated privileges.
**Resolution:** I established an isolated directory boundary using POSIX modes. The health directory is set to `root:kijanikiosk` mode `750`, while the inner JSON file is explicitly locked via `chown root:kijanikiosk` and `chmod 640`, granting safe, non-sudo reads to the group.

### Challenge C: Logrotate postrotate and PrivateTmp
**Conflict:** The `kk-logs` daemon utilizes `PrivateTmp=true`, operating in an isolated temporary namespace. Sequential `systemctl reload` triggers dispatched via standard `postrotate` hooks may fail when units lack dedicated `ExecReload` handlers.
**Resolution:** Given that the log aggregator is stateless and lightweight, I configured a `systemctl restart kk-logs.service` inside the logrotate postrotate script. This forces descriptors to gracefully drop and bind to newly rotated files.

### Challenge D: The Dirty VM and Package Holds
**Conflict:** Running infrastructure code on an environment containing previously installed and marked package holds can cause `apt-get` routines to error out or trigger unintended state downgrades.
**Resolution:** I coded an upfront state inspection conditional. The routine queries `apt-mark showhold` and bypasses package installation steps if the dependencies match, fulfilling pure idempotency.
